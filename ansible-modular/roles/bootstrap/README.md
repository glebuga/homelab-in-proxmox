# Роль: bootstrap

Одноразовая начальная настройка свежего хоста.

## Что она делает

1. Задаёт имя хоста и часовой пояс (`Europe/Moscow`).
2. Создаёт пользователя `admin`, добавляет его в группу sudo.
3. Создаёт `~/.ssh` и устанавливает ваш SSH-публичный ключ
   (`~/.ssh/id_ed25519.pub` на управляющей машине).
4. Выдаёт sudo без пароля через `/etc/sudoers.d/admin`.
5. Усиливает SSH: отключает вход по паролю, отключает вход root.
6. Блокирует пароль root и удаляет `authorized_keys` у root.

После выполнения хост доступен под `admin` с вашим SSH-ключом — без паролей.

## Переменные

| Переменная         | По умолчанию            | Где                  | Значение                                 |
|--------------------|-------------------------|----------------------|------------------------------------------|
| `admin_user`       | `admin`                 | `defaults/main.yml`  | Имя создаваемого admin-пользователя.     |
| `timezone`         | `Europe/Moscow`         | `defaults/main.yml`  | Часовой пояс системы.                    |
| `locale`           | `en_US.UTF-8`           | `defaults/main.yml`  | Локаль системы.                          |
| `ssh_public_keys`  | `id_ed25519.pub`        | `defaults/main.yml`  | Ключи, добавляемые admin-пользователю.   |
| `bootstrap_user`   | `root`                  | `defaults/main.yml`  | Пользователь для первого подключения.    |
| `sudo_group`       | `sudo` / `wheel`        | `vars/main.yml`      | Определяется по семейству ОС.            |
| `ssh_service`      | `ssh` / `sshd`          | `vars/main.yml`      | Определяется по семейству ОС.            |

## Handlers

- `restart ssh` — перезапускает SSH-сервис после изменения конфигурации.

## Как вызывается

Через [`playbooks/bootstrap.yml`](../playbooks/bootstrap.yml), который
устанавливает `ansible_user: root` (через `vars:` playbook'а), чтобы можно
было достучаться до хоста до создания пользователя admin.
